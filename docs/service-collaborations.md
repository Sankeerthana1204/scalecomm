# Service Collaboration Matrix

| Service | Main Responsibilities | Key API Operations | Collaborations | Interaction Type |
| --- | --- | --- | --- | --- |
| PatientService | Maintain patient profile lifecycle and patient lookup | `POST /api/v1/patients`, `GET /api/v1/patients/{patientId}`, `PUT /api/v1/patients/{patientId}`, `GET /api/v1/patients` | validates patient references for AppointmentService, HealthRecordService, BillingService, InsuranceService, PharmacyService | query, event |
| AppointmentService | Manage appointment booking and lifecycle transitions | `POST /api/v1/appointments`, `GET /api/v1/appointments/{appointmentId}`, `PUT /api/v1/appointments/{appointmentId}/reschedule`, `PUT /api/v1/appointments/{appointmentId}/cancel` | queries PatientService, emits booking lifecycle events to BillingService and NotificationService | query, command, event |
| HealthRecordService | Maintain clinical record history per patient | `POST /api/v1/records`, `GET /api/v1/records/{recordId}`, `GET /api/v1/patients/{patientId}/records` | queries PatientService, emits prescription-relevant updates to PharmacyService | query, event |
| BillingService | Issue invoices and track payments | `POST /api/v1/invoices`, `GET /api/v1/invoices/{invoiceId}`, `POST /api/v1/payments`, `GET /api/v1/payments/{paymentId}` | consumes appointment events, sends claim-related events to InsuranceService, sends billing status events to NotificationService | command, query, event |
| AuthService | Manage system users and roles | `POST /api/v1/users`, `GET /api/v1/users/{userId}`, `POST /api/v1/users/{userId}/roles`, `GET /api/v1/roles` | supports secured access for BFFs and emits user lifecycle events to NotificationService | command, query, event |
| PharmacyService | Track prescriptions and dispensation | `POST /api/v1/prescriptions`, `GET /api/v1/prescriptions/{prescriptionId}`, `POST /api/v1/prescriptions/{prescriptionId}/dispensations` | consumes health record events, queries PatientService, emits fulfillment events to NotificationService | command, query, event |
| InsuranceService | Manage policies and claims | `POST /api/v1/policies`, `GET /api/v1/policies/{policyId}`, `POST /api/v1/claims`, `GET /api/v1/claims/{claimId}` | queries PatientService, consumes billing events, emits claim result events to BillingService and NotificationService | command, query, event |
| NotificationService | Queue and track notifications | `POST /api/v1/notifications`, `GET /api/v1/notifications/{notificationId}`, `GET /api/v1/notifications/{notificationId}/status` | consumes events from AppointmentService, BillingService, AuthService, PharmacyService, InsuranceService | command, query, event |
